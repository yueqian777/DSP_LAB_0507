# TIMER API使用指南

## 1. 设备介绍

### 1.1 TIMER外设概述

本项目芯片支持的定时器外设包括：

- **定时器0**：延时函数delay，用于生成延时时间。
- **定时器1**：使用双计数模式，控制ADC和DAC的采样和输出。
- **定时器2**：本次接口使用的定时器，用于生成定时中断。
- **定时器3**：暂未使用。

### 1.2 TIMER外设应用原理

TIMER外设应用原理基于定时器2的控制。定时器开启后，每到指定时间间隔，会触发一次中断。用户可以根据此次中断来执行对应的操作，达到定时任务的效果。

## 2. 接口结构

### 2.1 头文件包含

使用TIMER API前，需要包含以下头文件：

```c
#include "timer_api.h"
```

### 2.2 宏定义
#### 2.2.1 定时器2频率宏定义

| 宏名称 | 值 | 说明 |
|-------|------|------|
| `TIMER_1HZ` | 10000 | 1Hz频率 |
| `TIMER_20HZ` | 20000 | 20Hz频率 |
| `TIMER_50HZ` | 50000 | 50Hz频率 |
| `TIMER_100HZ` | 100000 | 100Hz频率 |

### 2.3 全局变量

#### 2.3.1 定时器2标志位

| 变量名 | 类型 | 说明 |
|-------|------|------|
| `FLAG_TIM2` | `volatile unsigned char` | 定时器2中断标志位，定时器中断时置1 |

### 2.4 函数接口

#### 2.4.1 定时器2初始化函数

```c
void Tim2_Init(unsigned int freq);
```

**功能**：初始化定时器2，设置定时器频率和中断。

**参数**：
- `freq`：定时器频率，单位为Hz。

**使用场景**：在使用定时器前必须调用此函数，通常在系统初始化时调用。

## 3. 使用示例

### 3.1 基本使用示例

以下示例参考自 `user_timer.c` 文件，展示了如何初始化和使用定时器2。

定时器的使用方法是：
- 在程序初始化阶段，调用初始化定时器函数。
- 在主循环中检查定时器标志位是否为1，若为1则执行对应操作。
- 执行对应操作后，将定时器标志位设置为0，避免重复执行。

```c
#include "timer_api.h"          // 定时器用户接口
#include "led_api.h"            // LED用户接口
#include "system.h"             // 中断和延时初始化

// 定时器使用示例
void Timer_Example(void)
{
    // 初始化delay函数
    Sys_Init();
    // 初始化LED
    Led_Init();
    
    // 初始化定时器，设置为1Hz（1秒）
    Tim2_Init(TIMER_1HZ);

    for (;;)
    {
        // 检查定时器标志位是否为1
        if (FLAG_TIM2)
        {
            // 清除定时器标志位
            FLAG_TIM2 = 0;
            // 每秒钟切换所有LED的状态
            Led_Control(LED1_CORE, LED_TOGGLE);
            Led_Control(LED2_CORE, LED_TOGGLE);
        }
    }
}

```

### 3.2 多任务定时示例

以下示例展示了如何使用定时器实现多任务定时：

定时器多任务执行的方法是：
- 定义多个任务计数器，每个任务计数器对应一个任务。
- 在主循环中检查每个任务计数器是否超过任务执行时间。
- 若超过任务执行时间，执行对应操作并重置计数器。

```c
#include "timer_api.h"          // 定时器用户接口
#include "led_api.h"            // LED用户接口
#include "system.h"             // 中断和延时初始化

// 定时器使用示例2：多任务执行
void Timer_Example2(void)
{
    // 定义任务计数器
    unsigned int task1_count = 0;
    unsigned int task2_count = 0;

    // 初始化系统（开启中断）
    Sys_Init();
    // 初始化LED
    Led_Init();
    // 初始化定时器，设置为100Hz（10ms）
    Tim2_Init(100);

    for (;;)
    {
        // 检查定时器标志位
        if (FLAG_TIM2)
        {
            FLAG_TIM2 = 0;
            
            // 任务1：每1秒执行一次
            task1_count++;
            if (task1_count >= 100)  // 100 * 10ms = 1s
            {
                task1_count = 0;
                Led_Control(LED1_BLUE, LED_TOGGLE);
                Led_Control(LED1_RED, LED_TOGGLE);
            }
            
            // 任务2：每500ms执行一次
            task2_count++;
            if (task2_count >= 50)  // 50 * 10ms = 500ms
            {
                task2_count = 0;
                Led_Control(LED2_BLUE, LED_TOGGLE);
                Led_Control(LED2_RED, LED_TOGGLE);
            }
        }
    }
}

```

## 4. 问题解答

### 4.1 常见问题

#### Q1: 定时器中断不触发怎么办？

**A1**：请按照以下分层排查顺序进行检查：
- **应用层**：是否调用了 `Sys_Init()` 函数初始化中断功能？是否调用了 `Tim2_Init()` 函数初始化定时器？是否正确检查了标志位？
- **设备层**：请参考 `timer_api.h` 中的接口定义，确保API调用正确。
- **硬件层**：检查系统时钟配置是否正确，定时器是否被其他模块占用。

#### Q2: 定时器频率设置不准确怎么办？

**A2**：请按照以下分层排查顺序进行检查：
- **应用层**：检查初始化时传入的频率参数是否合理。
- **设备层**：请参考 `timer_api.h` 中的接口定义，确保API调用正确。
- **硬件层**：检查系统时钟频率是否与驱动中定义的一致。

#### Q3: 如何实现精确的延时功能？

**A3**：可以使用定时器实现精确的延时功能。例如：

```c
void Timer_Delay(unsigned int ms)
{
    unsigned int delay_count = 0;
    unsigned int target_count = ms / 10; // 假设定时器频率为100Hz
    
    FLAG_TIM2 = 0;
    
    while (delay_count < target_count) {
        if (FLAG_TIM2) {
            FLAG_TIM2 = 0;
            delay_count++;
        }
    }
}
```
备注：
该方法和系官方提供的延时函数”delay“实现方法类似。
因此在使用”delay“函数时，需要先调用 `Sys_Init()` 函数初始化中断功能和定时器0的功能。

#### Q4: 如何在定时器中断中执行复杂任务？

**A4**：定时器中断服务函数应该保持简短，避免执行复杂任务。对于复杂任务，可以在中断服务函数中设置标志位，然后在主循环中处理。


### 4.2 故障排查

- **应用层**：请基于 `timer_api.h` 和 `user_timer.c` 中的代码内容进行测试，确保API调用正确。
- **设备层**：如果遇到问题，请阅读驱动部分代码尝试排查或咨询工程师。
- **硬件层**：检查系统时钟配置和定时器硬件连接。

## 5. 总结

TIMER API提供了简单易用的接口，用于生成定时中断。通过使用 `Tim2_Init()` 函数初始化定时器，然后通过检查标志位来响应定时器中断事件，可以实现各种定时控制功能。

本指南介绍了TIMER设备的基本信息、API接口的使用方法和常见问题的解答，希望能够帮助开发者快速上手定时器控制功能，编写出高质量的应用程序。
