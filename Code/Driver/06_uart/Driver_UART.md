# UART驱动设计文档

## 1. 驱动架构

UART驱动采用分层架构设计，具体结构如下：

### 1.1 架构层次

| 模块       | 子模块           | 主要职责                   | 文件位置                                   |
| -------- | ------------- | ---------------------- | -------------------------------------- |
| **UART** | UART\_API     | UART用户接口模块，包含设备驱动和引脚控制 | `Driver/06_uart/uart_api.h`            |
| <br />   | UART\_EXAMPLE | UART应用示例模块，提供UART使用示例  | `User/user_driver/06_uart/user_uart.c` |

| 模块            | 子模块        | 主要职责                           | 文件位置                         |
| ------------- | ---------- | ------------------------------ | ---------------------------- |
| **UART\_API** | UART2\_DRV | UART2设备驱动模块，实现UART2的初始化和数据传输逻辑 | `Driver/06_uart/uart2_drv.c` |
| <br />        | UART2\_PIN | UART2引脚控制模块，负责UART2引脚的初始化和配置   | `Driver/06_uart/uart2_pin.c` |

### 1.2 模块关系

- **UART**：总模块，包含UART\_API和UART\_EXAMPLE子模块。
- **UART\_API**：用户接口模块，包含UART2\_DRV和UART2\_PIN子模块，为用户提供统一的控制接口。
- **UART2\_DRV**：设备驱动模块，实现UART2的初始化和数据传输逻辑，管理UART2的缓冲区和中断事件。
- **UART2\_PIN**：引脚控制模块，负责UART2引脚的初始化和配置，定义相关的时钟频率宏和引脚复用宏。
- **UART\_EXAMPLE**：应用示例模块，提供UART使用示例，展示如何初始化和使用UART。

## 2. 实现原理

### 2.1 芯片外设底层

#### 2.1.1 引脚配置

- **引脚复用**：通过 `SYSCFG_PINMUX4` 寄存器，将相关引脚配置为UART2功能。
- **TXD引脚**：配置为UART2的发送引脚。
- **RXD引脚**：配置为UART2的接收引脚。

#### 2.1.2 UART初始化

- **波特率设置**：根据用户指定的波特率，计算并设置UART的波特率寄存器。
- **FIFO配置**：设置UART的FIFO触发级别，支持1、4、8、14字节四种触发级别。
- **中断配置**：配置UART的中断，包括接收中断、发送中断和接收超时中断。

#### 2.1.3 数据传输

- **发送数据**：通过 `Uart2_SendBuffer` 函数，将数据写入发送缓冲区，等待中断服务函数发送。
- **接收数据**：通过中断服务函数，将接收到的数据写入接收缓冲区，设置相应的标志位。
- **缓冲区管理**：使用环形缓冲区管理接收数据，提高数据传输效率。

#### 2.1.4 中断处理

- **接收中断**：当UART接收缓冲区达到FIFO触发级别时，触发中断。将数据读取到接收缓冲区，设置相应的标志位FLAG_UART2_RX。
- **接收中断2**：当UART接收缓冲区达到FIFO触发级别时，触发中断。将数据读取到接收缓冲区，设置相应的标志位FLAG_UART2_RX_CTI。
- **发送中断**：当UART发送缓冲区为空时，触发中断。从发送缓冲区读取数据并继续发送。
- **接收超时中断**：当UART接收数据后一段时间没有新数据时，触发中断，处理不完整的数据包。

## 3. 总结

UART驱动采用分层架构设计，实现了从底层硬件到上层应用的完整控制流程。通过抽象UART接口和缓冲区管理，为用户提供了简单易用的API接口，同时保持了底层实现的灵活性和可扩展性。

驱动的核心设计思想是：

- **分层设计**：将硬件操作与应用逻辑分离，提高代码的可维护性和可扩展性。
- **抽象接口**：通过函数封装，为用户提供统一的控制接口，屏蔽底层实现细节。
- **中断驱动**：采用中断方式处理数据传输，提高系统的响应速度和实时性。
- **缓冲区管理**：使用环形缓冲区管理数据传输，提高数据传输效率。

这种设计不仅适用于UART控制，也可以推广到其他类似的串行通信外设控制场景，如SPI、I2C等。
