# DAC驱动设计文档

## 1. 驱动架构

DAC驱动采用分层架构设计，具体结构如下：

### 1.1 架构层次

| 模块 | 子模块 | 主要职责 | 文件位置 |
|------|---------|---------|----------|
| **DAC** | DAC_API | DAC用户接口模块，包含设备驱动、引脚控制和定时器控制 | `Driver/05_dac/dac_api.h` |
| | DAC_EXAMPLE | DAC应用示例模块，提供DAC使用示例 | `User/user_driver/05_dac/user_dac.c` |

| 模块 | 子模块 | 主要职责 | 文件位置 |
|------|---------|---------|----------|
| **DAC_API** | DAC_DRV | DAC设备驱动模块，实现DAC的底层驱动逻辑 | `Driver/05_dac/dac_drv.c` |
| | DAC_PIN | DAC引脚控制模块，负责DAC引脚的初始化和配置 | `Driver/05_dac/dac_pin.c` |
| | DAC_TIM1 | DAC定时器模块，实现DAC的定时输出控制 | `Driver/05_dac/dac_tim1.c` |
| | DAC_PRU | DAC PRU模块，实现DAC的并行输出控制 | `Driver/05_dac/dac_pru.c` |

### 1.2 模块关系

- **DAC**：总模块，包含DAC_API和DAC_EXAMPLE子模块。
- **DAC_API**：用户接口模块，包含DAC_DRV、DAC_PIN、DAC_TIM1和DAC_PRU子模块，为用户提供统一的控制接口。
- **DAC_DRV**：设备驱动模块，实现DAC的底层驱动逻辑，管理DAC的初始化和控制。
- **DAC_PIN**：引脚控制模块，负责DAC引脚的初始化和配置，定义相关的引脚宏。
- **DAC_TIM1**：定时器模块，实现DAC的定时输出控制，为DAC提供定时触发信号。
- **DAC_PRU**：PRU模块，实现DAC的并行输出控制，提高DAC输出的速度和效率。
- **DAC_EXAMPLE**：应用示例模块，提供DAC使用示例，展示如何初始化和使用DAC。

## 2. 实现原理

### 2.1 芯片外设底层

#### 2.1.1 时钟配置

- **定时器时钟**：定时器1，生成DAC输出触发信号，对应DAC输出频率。

PRU0实时检测TIM1低32位中断，检测后向DAC发送统一加载输出寄存器的信号。

#### 2.1.2 引脚配置

- **DAC_LOAD**：GPIO2[12]，DAC加载信号引脚。
- **DAC_CLR**：GPIO1[15]，DAC清除信号引脚。
- **SYS_SPI**：SPI外设，用于与DAC进行数据传输。

#### 2.1.3 其它配置

- **PRU0配置**：配置协处理器PRU0，用于配合TIM1低32位中断进行DAC的输出控制。

### 2.2 硬件设备驱动层

#### 2.2.1 DAC初始化

- **输出频率设置**：根据用户指定的输出频率，计算并设置定时器1低32位中断的定时周期。
- **输出长度设置**：根据用户指定的输出长度，配置缓冲区大小。
- **通道选择**：根据用户指定的通道选择，配置DAC的输出通道。（减少输出通道数，可少量增加输出频率，待测试。）

#### 2.2.2 数据输出

- **Ping-Pong缓冲区**：使用双缓冲区（Ping-Pong）机制，实现数据的连续输出和更新。
- **数据传输**：PRU0通过SPI外设将数据传输到DAC。
- **定时触发**：通过定时器1定时触发，PRU0生成DAC_LOAD统一加载信号。
- **并行输出**：DAC_LOAD信号触发DAC加载数据寄存器，实现并行输出。

#### 2.2.3 标志位生成

- **FLAG_DA**：当PRU0按照定时器的频率，输出完指定长度的数据时，更新缓冲区标志位，通知应用层更新数据。

## 3. 总结

DAC驱动采用分层架构设计，实现了从底层硬件到上层应用的完整控制流程。通过抽象DAC接口和使用PRU驱动DAC输出，为用户提供了高效的DAC数据输出功能。

驱动的核心设计思想是：
- **分层设计**：将硬件操作与应用逻辑分离，提高代码的可维护性和可扩展性。
- **抽象接口**：通过函数封装，为用户提供统一的控制接口，屏蔽底层实现细节。
- **PRU0驱动**：使用PRU0实现DAC的控制输出，提高输出速度和效率。
- **Ping-Pong缓冲区**：使用双缓冲区机制，实现数据的连续输出和更新。
- **定时控制**：通过定时器实现DAC的定时输出，确保输出的准确性和稳定性。
