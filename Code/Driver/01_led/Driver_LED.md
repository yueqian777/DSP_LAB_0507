# LED驱动设计文档

## 1. 驱动架构

LED驱动采用分层架构设计，具体结构如下：

### 1.1 架构层次

| 模块 | 子模块 | 主要职责 | 文件位置 |
|------|---------|---------|----------|
| **LED** | LED_API | LED用户接口模块，包含设备驱动和引脚控制 | `Driver/01_led/led_api.h` |
| | LED_EXAMPLE | LED应用示例模块，提供LED使用示例 | `User/user_driver/01_led/user_led.c` |

| 模块 | 子模块 | 主要职责 | 文件位置 |
|------|---------|---------|----------|
| **LED_API** | LED_DRV | LED设备驱动模块，实现LED的初始化和控制逻辑 | `Driver/01_led/led_drv.c` |
| | LED_PIN | LED引脚控制模块，负责LED引脚的初始化和状态设置 | `Driver/01_led/led_pin.c` |

### 1.2 模块关系

- **LED**：总模块，包含LED_API和LED_EXAMPLE子模块。
- **LED_API**：用户接口模块，包含LED_DRV和LED_PIN子模块，为用户提供统一的控制接口。
- **LED_DRV**：设备驱动模块，实现LED的初始化和控制逻辑，维护LED索引到实际引脚的映射。
- **LED_PIN**：引脚控制模块，直接操作硬件寄存器，实现底层的GPIO控制。
- **LED_EXAMPLE**：应用示例模块，提供LED使用示例，展示如何初始化和控制LED。

## 2. 实现原理

### 2.1 芯片外设底层

#### 2.1.1 引脚定义与配置

- **引脚宏定义**：在 `led_pin.h` 中定义了所有LED相关的引脚宏，包括引脚编号、掩码和使能值。
- **PSC初始化**：通过 `PSCModuleControl` 函数使能GPIO模块的电源。
- **引脚复用配置**：通过操作 `SYSCFG_PINMUX` 寄存器，将LED相关引脚配置为GPIO功能。
- **GPIO初始化**：通过 `GPIODirModeSet` 函数将LED引脚设置为输出模式。

#### 2.1.2 底层控制实现

- **LED状态控制**：通过 `GPIOPinWrite` 函数直接操作GPIO寄存器，实现LED的点亮、关闭和翻转操作。
- **状态翻转实现**：通过 `GPIOPinRead` 函数读取当前引脚状态，然后取反写入，实现LED状态的翻转。

### 2.2 硬件设备驱动层

#### 2.2.1 LED索引映射

- **LED索引表**：在 `led_drv.c` 中定义了 `ledPinTable` 数组，将LED索引（如 `LED1_BLUE`）映射到实际的引脚编号。
- **索引验证**：在控制LED时，首先验证LED索引是否在有效范围内，确保操作的安全性。

#### 2.2.2 驱动逻辑实现

- **初始化流程**：`Led_Init` 函数调用 `LED_PinInit` 函数，完成引脚的初始化和配置。
- **控制流程**：`Led_Control` 函数根据LED索引和状态，查找对应的引脚编号，然后调用 `LED_PinSet` 函数控制引脚状态。

## 3. 总结

LED驱动采用分层架构设计，实现了从底层硬件到上层应用的完整控制流程。通过抽象LED索引和状态，为用户提供了简单易用的API接口，同时保持了底层实现的灵活性和可扩展性。

驱动的核心设计思想是：
- **分层设计**：将硬件操作与应用逻辑分离，提高代码的可维护性和可扩展性。
- **抽象接口**：通过枚举和函数封装，为用户提供统一的控制接口，屏蔽底层实现细节。
- **映射机制**：通过索引映射表，将抽象的LED索引映射到实际的硬件引脚，实现灵活的硬件配置。

这种设计不仅适用于LED控制，也可以推广到其他类似的外设控制场景，如按键、蜂鸣器等。
