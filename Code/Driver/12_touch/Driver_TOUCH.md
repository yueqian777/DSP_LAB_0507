# TOUCH驱动设计文档

## 1. 驱动架构

TOUCH驱动采用分层架构设计，具体结构如下：

### 1.1 架构层次

| 模块 | 子模块 | 主要职责 | 文件位置 |
|------|---------|---------|----------|
| **TOUCH** | TOUCH_API | Touch用户接口模块，包含设备驱动、引脚控制、扫描和按钮等 | `Driver/12_touch/touch_api.h` |
| | TOUCH_EXAMPLE | Touch应用示例模块，提供Touch使用示例 | `User/user_driver/12_touch/user_touch.c` |

| 模块 | 子模块 | 主要职责 | 文件位置 |
|------|---------|---------|----------|
| **TOUCH_API** | TOUCH_DRV | Touch设备驱动模块，实现Touch的初始化和GT1151芯片控制 | `Driver/12_touch/touch_drv.c` |
| | TOUCH_PIN | Touch引脚控制模块，负责Touch引脚的初始化和中断配置 | `Driver/12_touch/touch_pin.c` |
| | TOUCH_SCAN | Touch扫描模块，实现触摸屏扫描功能 | `Driver/12_touch/touch_scan.c` |
| | TOUCH_BUTTON | Touch按钮模块，处理UI按钮的触摸响应 | `Driver/12_touch/touch_btn.c` |

### 1.2 模块关系

- **TOUCH**：总模块，包含TOUCH_API和TOUCH_EXAMPLE子模块。
- **TOUCH_API**：用户接口模块，包含TOUCH_DRV、TOUCH_PIN、TOUCH_SCAN和TOUCH_BUTTON子模块，为用户提供统一的控制接口。
- **TOUCH_DRV**：设备驱动模块，实现Touch的初始化逻辑，控制GT1151触摸芯片。
- **TOUCH_PIN**：引脚控制模块，负责Touch引脚的初始化和中断配置，定义相关的引脚宏。
- **TOUCH_SCAN**：扫描模块，实现触摸屏扫描功能，读取触摸坐标和状态。
- **TOUCH_BUTTON**：按钮模块，处理UI按钮的触摸响应，更新按钮标志位。
- **TOUCH_EXAMPLE**：应用示例模块，提供Touch使用示例，展示如何初始化和使用Touch。

## 2. 实现原理

### 2.1 芯片外设底层

#### 2.1.1 引脚配置

- **TOUCH_INT**：触摸中断引脚，用于检测触摸事件。
- **I2C接口**：用于与GT1151触摸芯片通信。
- **引脚复用配置**：通过操作`SYSCFG_PINMUX`寄存器，将Touch相关引脚配置为GPIO或I2C功能。
- **中断配置**：配置触摸中断为下降沿触发，实现触摸事件的实时检测。

#### 2.1.2 I2C通信

- **I2C初始化**：配置I2C总线的时钟频率和通信参数。
- **寄存器读写**：通过I2C接口读写GT1151芯片的寄存器。
- **数据传输**：使用I2C进行触摸坐标数据的读取。

#### 2.1.3 GT1151芯片控制

- **芯片初始化**：配置GT1151的工作模式和参数。
- **状态读取**：读取触摸状态寄存器，判断是否有触摸发生。
- **坐标读取**：读取触摸坐标寄存器，获取触摸点的X、Y坐标。

### 2.2 硬件设备驱动层

#### 2.2.1 Touch初始化流程

1. **引脚初始化**：调用`Touch_Pin_Init_Out`函数，配置触摸中断引脚为输出模式。
2. **芯片复位**：通过GPIO引脚复位GT1151芯片。
3. **引脚重新配置**：调用`Touch_Pin_Init_In`函数，配置触摸中断引脚为输入模式，并配置中断。
4. **I2C初始化**：配置I2C总线，用于与GT1151通信。
5. **芯片初始化**：配置GT1151的工作参数。

#### 2.2.2 触摸扫描实现

- **中断触发**：当触摸发生时，GT1151触发中断引脚。
- **中断处理**：中断服务函数设置触摸标志位。
- **坐标读取**：应用层检测到触摸标志后，调用扫描函数读取触摸坐标。
- **状态更新**：更新触摸状态变量和按钮标志位。

#### 2.2.3 按钮响应实现

- **触摸坐标判断**：根据触摸坐标判断是否点击了某个按钮。
- **按钮消息处理**：使用grlib控件库处理按钮消息。
- **标志位更新**：当按钮被点击时，更新相应的按钮标志位。

## 3. 总结

TOUCH驱动采用分层架构设计，实现了从底层硬件到上层应用的完整控制流程。通过抽象Touch接口和使用中断机制，为用户提供了高效、实时的触摸交互功能。

驱动的核心设计思想是：

- **分层设计**：将硬件操作与应用逻辑分离，提高代码的可维护性和可扩展性。
- **抽象接口**：通过函数封装，为用户提供统一的控制接口，屏蔽底层实现细节。
- **中断驱动**：使用中断机制实现触摸事件的实时检测，减少CPU占用。
- **I2C通信**：通过I2C接口与GT1151芯片通信，实现触摸数据的读取。
- **控件集成**：集成grlib控件库，支持按钮等UI控件的触摸响应。
