# RS485驱动设计文档

## 1. 驱动架构

RS485驱动采用分层架构设计，具体结构如下：

### 1.1 架构层次

| 模块       | 子模块           | 主要职责                   | 文件位置                                   |
| -------- | ------------- | ---------------------- | -------------------------------------- |
| **RS485** | RS485\_API    | RS485用户接口模块，包含设备驱动和引脚控制 | `Driver/07_rs485/rs485_api.h`            |
| <br />   | RS485\_EXAMPLE | RS485应用示例模块，提供RS485使用示例  | `User/user_driver/07_rs485/user_rs485.c` |

| 模块            | 子模块         | 主要职责                           | 文件位置                         |
| ------------- | ----------- | ------------------------------ | ---------------------------- |
| **RS485\_API** | RS485\_DRV  | RS485设备驱动模块，实现RS485的初始化和数据传输逻辑 | `Driver/07_rs485/rs485_drv.c` |
| <br />        | RS485\_PIN  | RS485引脚控制模块，负责RS485引脚的初始化和配置   | `Driver/07_rs485/rs485_pin.c` |
| <br />        | UART1\_DRV  | UART1设备驱动模块，提供RS485的串行通信支持 | `Driver/07_rs485/uart1_drv.c` |
| <br />        | UART1\_PIN  | UART1引脚控制模块，负责UART1引脚的初始化和配置   | `Driver/07_rs485/uart1_pin.c` |

### 1.2 模块关系

- **RS485**：总模块，包含RS485\_API和RS485\_EXAMPLE子模块。
- **RS485\_API**：用户接口模块，包含RS485\_DRV、RS485\_PIN、UART1\_DRV和UART1\_PIN子模块，为用户提供统一的控制接口。
- **RS485\_DRV**：设备驱动模块，实现RS485的初始化和数据传输逻辑，管理RS485的缓冲区和中断事件。
- **RS485\_PIN**：引脚控制模块，负责RS485引脚的初始化和配置，定义相关的使能引脚和方向引脚。
- **UART1\_DRV**：UART1设备驱动模块，提供RS485的串行通信支持，实现UART1的初始化和数据传输。
- **UART1\_PIN**：UART1引脚控制模块，负责UART1引脚的初始化和配置，定义相关的时钟频率宏和引脚复用宏。
- **RS485\_EXAMPLE**：应用示例模块，提供RS485使用示例，展示如何初始化和使用RS485。

## 2. 实现原理

### 2.1 芯片外设底层

#### 2.1.1 引脚配置

- **引脚复用**：通过 `SYSCFG_PINMUX1` 寄存器，将相关引脚配置为UART1功能。
- **TXD引脚**：配置为UART1的发送引脚。
- **RXD引脚**：配置为UART1的接收引脚。
- **DE/RE引脚**：配置为GPIO引脚，用于控制RS485的发送/接收模式切换。

#### 2.1.2 UART1初始化

- **波特率设置**：根据用户指定的波特率，计算并设置UART1的波特率寄存器。
- **FIFO配置**：设置UART1的FIFO触发级别，支持1、4、8、14字节四种触发级别。
- **中断配置**：配置UART1的中断，包括接收中断、发送中断和接收超时中断。

#### 2.1.3 RS485模式控制

- **发送模式**：通过控制DE/RE引脚，将RS485芯片设置为发送模式。
- **接收模式**：通过控制DE/RE引脚，将RS485芯片设置为接收模式。

#### 2.1.4 数据传输

- **发送数据**：通过 `Rs485_SendBuffer` 函数，将数据写入发送缓冲区，等待中断服务函数发送。
- **接收数据**：通过中断服务函数，将接收到的数据写入接收缓冲区，设置相应的标志位。
- **缓冲区管理**：使用环形缓冲区管理接收数据，提高数据传输效率。

#### 2.1.5 中断处理

- **接收中断**：当UART1接收缓冲区达到FIFO触发级别时，触发中断。将数据读取到接收缓冲区，设置相应的标志位FLAG_UART1_RX。
- **接收超时中断**：当UART1接收数据后一段时间没有新数据时，触发中断，处理不完整的数据包，设置相应的标志位FLAG_UART1_RX_CTI。
- **发送中断**：当UART1发送缓冲区为空时，触发中断。从发送缓冲区读取数据并继续发送。
- **发送完成中断**：当全部数据发送完成后，设置发送完成标志位FLAG_UART1_TX。

## 3. 总结

RS485驱动采用分层架构设计，实现了从底层硬件到上层应用的完整控制流程。通过抽象RS485接口和缓冲区管理，为用户提供了简单易用的API接口，同时保持了底层实现的灵活性和可扩展性。

驱动的核心设计思想是：

- **分层设计**：将硬件操作与应用逻辑分离，提高代码的可维护性和可扩展性。
- **抽象接口**：通过函数封装，为用户提供统一的控制接口，屏蔽底层实现细节。
- **中断驱动**：采用中断方式处理数据传输，提高系统的响应速度和实时性。
- **缓冲区管理**：使用环形缓冲区管理数据传输，提高数据传输效率。
- **半双工通信**：通过控制DE/RE引脚，实现RS485的半双工通信模式切换。

这种设计不仅适用于RS485控制，也可以推广到其他类似的串行通信外设控制场景，如CAN、Modbus等。
