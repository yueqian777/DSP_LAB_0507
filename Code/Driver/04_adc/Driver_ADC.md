# ADC驱动设计文档

## 1. 驱动架构

ADC驱动采用分层架构设计，具体结构如下：

### 1.1 架构层次

| 模块           | 子模块          | 主要职责                               | 文件位置                       |
| ------------ | ------------ | ---------------------------------- | -------------------------- |
| **ADC**      | ADC\_API     | ADC用户接口模块，包含设备驱动、引脚控制、EMIF和EDMA传输等 | `Driver/04_adc/adc_api.h`  |
| <br />       | ADC\_EXAMPLE | ADC应用示例模块，提供ADC使用示例                | `User/user_driver/04_adc/user_adc.c` |

| 模块 | 子模块 | 主要职责 | 文件位置 |
|------|---------|---------|----------|
| **ADC\_API** | ADC\_DRV     | ADC设备驱动模块，实现ADC的底层驱动逻辑             | `Driver/04_adc/adc_drv.c`  |
| <br />       | ADC\_PIN     | ADC引脚控制模块，负责ADC引脚的初始化和配置           | `Driver/04_adc/adc_pin.c`  |
| <br />       | ADC\_EMIF    | EMIF模块，负责ADC数据的EMIF传输              | `Driver/04_adc/adc_emif.c` |
| <br />       | ADC\_EDMA    | EDMA模块，实现ADC数据的DMA传输               | `Driver/04_adc/adc_edma.c` |
| <br />       | ADC\_TIM1    | TIM1定时器模块，实现ADC的定时采样控制             | `Driver/04_adc/adc_tim1.c` |
| <br />       | ADC\_PRU1    | 协处理器PRU1模块，配合TIM1进行ADC的电压转换驱动      | `Driver/04_adc/adc_pru1.c` |

### 1.2 模块关系

- **ADC**：总模块，包含ADC\_API和ADC\_EXAMPLE子模块。
- **ADC\_API**：用户接口模块，包含ADC\_DRV、ADC\_PIN和ADC\_TIM1等子模块，为用户提供统一的控制接口。
- **ADC\_DRV**：设备驱动模块，实现ADC的底层驱动逻辑，管理ADC的初始化和控制。
- **ADC\_PIN**：引脚控制模块，负责ADC引脚的初始化和配置，定义相关的引脚宏。
- **ADC\_EMIF**：EMIF模块，负责ADC数据的EMIF传输，为ADC提供EMIF接口。
- **ADC\_EDMA**：EDMA模块，实现ADC数据的DMA传输，提高数据传输效率。
- **ADC\_TIM1**：TIM1定时器模块，实现ADC的定时采样控制，为ADC提供定时触发信号。
- **ADC\_PRU1**：协处理器PRU1模块，配合TIM1进行ADC的电压转换驱动。
- **ADC\_EXAMPLE**：应用示例模块，提供ADC使用示例，展示如何初始化和使用ADC。


## 2. 实现原理

### 2.1 芯片外设底层

#### 2.1.1 时钟配置

- **定时器时钟**：定时器1，生成ADC采样触发信号，对应ADC采集频率。

PRU1实时检测TIM1高32位中断，检测后向ADC发送转换开始信号。

#### 2.1.2 引脚配置

- **ADC\_BUSY**：GPIO5\[11]，ADC忙信号引脚，指示ADC是否正在转换数据。用于触发EDMA传输一组EMIF数据。
- **ADC\_CONVST**：GPIO5\[13]，ADC转换开始引脚，触发ADC转换开始。在PRU1中检测TIM1中断，生成ADC转换开始信号。
- **ADC\_RESET**：GPIO5\[12]，ADC复位引脚。
- **EMIF配置**：EMIFA+CS2，配置EMIF通道，将ADC采集的数据传输到EMIF缓冲区。

#### 2.1.3 其它配置

- **EDMA配置**：配置EDMA通道，将EMIF缓冲区中的数据传输到用户缓冲区。
- **PRU1配置**：配置协处理器PRU1，用于配合TIM1高32位中断进行ADC的电压转换驱动。

### 2.2 硬件设备驱动层

#### 2.2.1 ADC初始化

- **采样频率设置**：根据用户指定的采样频率，计算并设置定时器1高32位中断的定时周期。
- **采样长度设置**：根据用户指定的采样长度，配置缓冲区大小。

#### 2.2.2 数据采集

- **定时触发**：通过定时器1定时触发，PRU1生成ADC_CONVST转换开始信号。
- **数据传输**：使用ADC_BUSY引脚触发EDMA，EDMA从EMIF缓冲区读取数据，传输到用户缓冲区。
- **Ping-Pong缓冲区**：使用双缓冲区（Ping-Pong）机制，实现数据的连续采集和处理。

#### 2.2.3 标志位生成

- **FLAG_AD**：当EDMA传输完成一帧数据时，触发中断，更新缓冲区标志位，通知应用层更新数据。

## 3. 总结

ADC驱动采用分层架构设计，实现了从底层硬件到上层应用的完整控制流程。通过抽象ADC接口和使用EDMA传输，为用户提供了高效的ADC数据采集功能。

驱动的核心设计思想是：

- **分层设计**：将硬件操作与应用逻辑分离，提高代码的可维护性和可扩展性。
- **抽象接口**：通过函数封装，为用户提供统一的控制接口，屏蔽底层实现细节。
- **EDMA传输**：使用EDMA进行数据传输，提高数据传输效率，减少CPU占用。
- **PRU1驱动**：通过协处理器PRU1，配合TIM器1实现ADC的电压转换驱动，减少CPU占用。
- **Ping-Pong缓冲区**：使用双缓冲区机制，实现数据的连续采集和处理。
- **定时控制**：通过定时器实现ADC的定时采样，确保采样的准确性和稳定性。
