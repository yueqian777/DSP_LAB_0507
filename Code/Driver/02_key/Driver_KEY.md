# KEY驱动设计文档

## 1. 驱动架构

KEY驱动采用分层架构设计，具体结构如下：

### 1.1 架构层次

| 模块 | 子模块 | 主要职责 | 文件位置 |
|------|---------|---------|----------|
| **KEY** | KEY_API | 按键用户接口模块，包含设备驱动和引脚控制 | `Driver/02_key/key_api.h` |
| | KEY_EXAMPLE | 按键应用示例模块，提供按键使用示例 | `User/user_driver/02_key/` |

| 模块 | 子模块 | 主要职责 | 文件位置 |
|------|---------|---------|----------|
| **KEY_API** | KEY_DRV | 按键设备驱动模块，实现按键的初始化和中断处理逻辑 | `Driver/02_key/key_drv.c` |
| | KEY_PIN | 按键引脚控制模块，负责按键引脚的初始化和中断设置 | `Driver/02_key/key_pin.c` |

### 1.2 模块关系

- **KEY**：总模块，包含KEY_API和KEY_EXAMPLE子模块。
- **KEY_API**：用户接口模块，包含KEY_DRV和KEY_PIN子模块，为用户提供统一的控制接口。
- **KEY_DRV**：设备驱动模块，实现按键的初始化和中断处理逻辑，维护按键状态的标志位。
- **KEY_PIN**：引脚控制模块，直接操作硬件寄存器，实现底层的GPIO控制和中断配置。
- **KEY_EXAMPLE**：应用示例模块，提供按键使用示例，展示如何初始化和使用按键。

## 2. 实现原理

### 2.1 芯片外设底层

#### 2.1.1 引脚定义与配置

- **引脚宏定义**：在 `key_pin.h` 中定义了所有按键相关的引脚宏，包括引脚编号、掩码和使能值。
- **PSC初始化**：通过 `PSCModuleControl` 函数使能GPIO模块的电源。
- **引脚复用配置**：通过操作 `SYSCFG_PINMUX` 寄存器，将按键相关引脚配置为GPIO功能。
- **GPIO初始化**：通过 `GPIODirModeSet` 函数将按键引脚设置为输入模式。

#### 2.1.2 中断配置与处理

- **中断触发方式**：配置按键引脚为下降沿触发中断，当按键按下时产生中断。
- **中断使能**：通过 `GPIOIntEnable` 函数使能按键引脚的中断。
- **中断服务函数**：注册按键中断服务函数，处理按键按下事件。

### 2.2 硬件设备驱动层

#### 2.2.1 按键状态管理

- **标志位管理**：定义了 `FLAG_KEY1` 到 `FLAG_KEY5` 五个全局标志位，用于标识按键的按下状态。
- **中断处理**：在中断服务函数中，根据中断源设置对应的按键标志位。

#### 2.2.2 驱动逻辑实现

- **初始化流程**：`Key_Init` 函数调用 `KEY_PinInit` 函数，完成引脚的初始化和中断配置。
- **中断处理流程**：当按键按下时，触发GPIO中断，中断服务函数根据中断源设置对应的按键标志位。

## 3. 总结

KEY驱动采用分层架构设计，实现了从底层硬件到上层应用的完整控制流程。通过抽象按键标志位和初始化函数，为用户提供了简单易用的API接口，同时保持了底层实现的灵活性和可扩展性。

驱动的核心设计思想是：
- **分层设计**：将硬件操作与应用逻辑分离，提高代码的可维护性和可扩展性。
- **抽象接口**：通过标志位和函数封装，为用户提供统一的控制接口，屏蔽底层实现细节。
- **中断驱动**：采用中断方式处理按键事件，提高系统的响应速度和实时性。

这种设计不仅适用于按键控制，也可以推广到其他类似的外设控制场景，如触摸按键、旋钮等。
