# 项目 3.2 触摸界面设计

## 目标与边界

在现有 C6748 实时音频主循环上增加轻量 LCD/Touch 控制界面。界面只写模式请求和 codec 码率请求，不直接调用 WOLA、降噪或 codec 的复位/处理函数，也不在中断或 `Subband_Process_1024()` 内绘屏。

## 硬件依据

- LCD：`LCD_WIDTH=800`、`LCD_HEIGHT=480`，RGB565 离屏缓冲。
- GT1151：配置表分辨率字节为 `0x0320 × 0x01E0`，即 800×480；驱动直接输出 `Touch_X/Touch_Y`，有效布局范围按 X=0..799、Y=0..479。
- 字体：当前 grlib 只链接 ASCII 字体。固定中文标题和状态词采用少量 16×16 静态只读字模；数字和调试数据使用现有 ASCII 字体。

## 架构

- `user_subband_ui_logic.c/.h`：纯状态逻辑，包含按钮到 Mode 映射、码率合法化、codec 模式判断、倒计时计算和按下-释放锁，可在 PC 上单测。
- `user_subband_ui.c/.h`：LCD 初始化、八个模式 Widget、三码率矩形命中、dirty-region 绘制和 Watch 调试量。
- `user_subband_flow.c/.h`：仅增加持久码率控制层和主循环 UI 调度。Mode 8/11 进入时读取持久目标；运行中切码率只写 `SUBBAND_CODEC_LOOP_DebugRequestedTargetKbps`。

## 固定布局

| 区域 | 坐标 |
|---|---|
| 标题 | (0,0)-(799,39) |
| 状态 | (16,44)-(783,115) |
| 八个模式按钮 | 四列两行，X=16/212/408/604，Y=124/188，W=180，H=52 |
| 三个码率按钮 | X=116/316/516，Y=258，W=168，H=46 |
| 学习状态与倒计时 | (16,316)-(783,383) |
| 进度条 | (24,390)-(775,409) |
| 算法负载 | (16,420)-(783,463) |

模式按钮依次映射 0、1、2、4、6、7、8、11。Mode 3、5、9、10 保留但不显示。码率默认 240 kbps，只在 Mode 8/11 激活。

## 触摸与刷新

`FLAG_TOUCH` 到来后先清标志，再调用 `Touch_Scan()`。八个 Widget 的 release callback 置 `FLAG_BUTTON_1..8`；码率区域使用 `Touch_X/Touch_Y` 命中。锁存器只接受一次按下，必须观察到释放后才接受下一次。点击已应用 Mode 不重复请求。

静态背景只初始化一次。动态区域使用绝对坐标局部清除；模式、状态、倒计时、码率、负载分别设置 dirty bit。每次 Service 最多刷新两个区域，且只在无待处理 AD/DA 事件时调用。

## 验证

- PC 逻辑测试：8 模式映射、码率合法化、五种学习模式、倒计时、长按单触发。
- CCS 构建：Debug/O3、link_errors=0、记录 section 增量。
- DSS 板测：注入八个按钮标志和码率请求，验证 Applied Mode、学习不因动态切码率复位、帧持续增长、UI 计时变量。
- 无法由 DSS 证明的肉眼显示和主观音频项目明确标记未测。
